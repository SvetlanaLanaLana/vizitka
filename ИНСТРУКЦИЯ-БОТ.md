# Инструкция: бот для заявок с сайта

Заявки с формы на сайте отправляются в ваш Telegram через бота.  
Ниже — как создать **своего** бота и подключить его к сайту (или заменить на другого).

---

## Часть 1. Создать бота в Telegram

### Шаг 1. Откройте BotFather
1. В Telegram найдите [@BotFather](https://t.me/BotFather)
2. Нажмите **Start** или отправьте `/start`

### Шаг 2. Создайте нового бота
1. Отправьте команду: `/newbot`
2. Введите **название** бота (как видят пользователи), например:  
   `Заявки — Светлана Подольская`
3. Введите **username** (латиница, обязательно оканчивается на `bot`), например:  
   `zayavka_vizitka_bot`

### Шаг 3. Сохраните токен
BotFather пришлёт сообщение с **токеном**, например:
```
8551650843:AAHxxxxxxxxxxxxxxxxxxxxxxxx
```

**Важно:** токен — это пароль бота. Никому не показывайте и не публикуйте в интернете.

---

## Часть 2. Узнать ваш chat_id (куда приходят заявки)

Заявки с сайта приходят **вам в личный Telegram**, не в чат с клиентом.

1. Напишите [@userinfobot](https://t.me/userinfobot) → он пришлёт ваш **Id** (число)
2. Напишите **своему новому боту** `/start` (обязательно!)
3. Скопируйте число из userinfobot — это ваш `chatId`

## Часть 3. Настроить сайт

### Вариант А: GitHub Pages (svetlanalanalana.github.io)

На GitHub Pages **нет PHP** — `send-telegram.php` там не работает. Нужен бесплатный API на **Vercel**:

1. Зарегистрируйтесь на [vercel.com](https://vercel.com) через GitHub
2. **Add New → Project** → выберите репозиторий `vizitka`
3. В **Environment Variables** добавьте:
   - `TELEGRAM_BOT_TOKEN` — токен от BotFather
   - `TELEGRAM_CHAT_ID` — ваш Id (например `5232226311`)
   - `TELEGRAM_BOT_USERNAME` — `zayavka_vizitka_bot`
4. Нажмите **Deploy**
5. Скопируйте URL проекта, например: `https://vizitka-xxxx.vercel.app`
6. В файле **`form-config.js`** вставьте:
   ```js
   var vercelApiUrl = 'https://vizitka-xxxx.vercel.app/api/send-telegram';
   ```
7. Загрузите обновлённый `form-config.js` на GitHub Pages

Сайт остаётся на GitHub Pages, заявки уходят через Vercel API.

**Альтернатива:** развернуть **весь сайт** на Vercel (не только API) — тогда `form-config.js` менять не нужно, URL будет `/api/send-telegram` автоматически.

---

### Вариант Б: хостинг с PHP

Откройте файл **`telegram-config.php`** и вставьте данные:

```php
<?php
return [
    'botToken' => 'ВАШ_ТОКЕН_ОТ_BOTFATHER',
    'chatId' => 'ВАШ_CHAT_ID',
    'botUsername' => 'zayavka_sveta_bot',  // username без @
];
```

Сохраните файл и загрузите на хостинг вместе с сайтом.

**Обязательные файлы на хостинге:**
- `index.html`
- `form.js`
- `send-telegram.php`
- `telegram-config.php`

Сайт должен открываться через **хостинг** (не двойным кликом по файлу на компьютере).

## Часть 4. Ссылка на бота на сайте

Если хотите, чтобы посетители писали **напрямую в бота** (не только через форму), используйте ссылку:

```
https://t.me/ИМЯ_БОТА
```

Пример для бота `zayavka_sveta_bot`:
```
https://t.me/zayavka_sveta_bot
```

### Где менять ссылки на сайте

В файле **`index.html`** найдите имя бота и замените на **вашего** (сейчас: `zayavka_vizitka_bot`):

| Место | Что заменить |
|-------|----------------|
| Блок «Контакты» | ссылка Telegram |
| Футер | ссылка Telegram |
| `privacy.html` | упоминание бота |

**Пример замены:**
```html
<!-- Было -->
<a href="https://t.me/zayavka_vizitka_bot">@zayavka_vizitka_bot</a>

<!-- Стало -->
<a href="https://t.me/zayavka_sveta_bot">@zayavka_sveta_bot</a>
```

Форма заявки отправляет сообщения через `send-telegram.php` — для неё достаточно обновить **`telegram-config.php`** (токен и chat_id).  
Ссылки `t.me/...` нужны, если хотите отдельную кнопку «Написать в Telegram».

## Часть 5. Проверка

1. GitHub Pages: настройте Vercel API и `form-config.js`. PHP-хостинг: загрузите `telegram-config.php`
2. Напишите боту `/start`
3. Откройте сайт → **Оставить заявку**
4. Заполните форму (телефон **или** email)
5. Заявка должна прийти в Telegram от имени бота

Если не пришла — проверьте:
- правильный `chatId` в `telegram-config.php`
- вы написали боту `/start`
- сайт на хостинге, а не `file://` с компьютера
- в `telegram-config.php` нет лишних пробелов в токене

## Два бота: зачем и как

| Бот | Задача |
|-----|--------|
| **Бот для заявок с сайта** | Принимает заявки из формы (через токен в `telegram-config.php`) |
| **Другой бот (ссылка на сайте)** | Клиенты пишут напрямую, например для консультаций |

Можно использовать **одного** бота и для формы, и для ссылки — укажите одно имя везде.

Если нужны **два разных**:
1. Создайте второго бота в BotFather (`/newbot`)
2. В `telegram-config.php` — токен бота, который **принимает заявки с формы**
3. В ссылках на сайте — `https://t.me/второй_бот` для общения с клиентами

---

## Безопасность

- Не выкладывайте `telegram-config.php` в открытый репозиторий GitHub (файл в `.gitignore`)
- На Vercel токен храните только в **Environment Variables**, не в коде
- Если токен утёк — в BotFather: `/mybots` → ваш бот → **Revoke token** → получите новый

---

## Краткая шпаргалка

```
1. @BotFather → /newbot → имя + username
2. Скопировать токен
3. @userinfobot → узнать chat_id
4. Написать боту /start
5. Вставить в telegram-config.php
6. Загрузить сайт на хостинг
7. Ссылка на бота: https://t.me/ваш_бот
```
